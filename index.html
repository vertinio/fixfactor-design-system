Design Philosophy

The Fixfactor Design System is built on the following foundations that define our visual identity and user experience:

  1. Clarity and Functionality - each element has a clear purpose and is intuitive to use
  2. Professionalism and Trust - design reflects the expert nature of the company and builds confidence
  3. Bauhaus meets Apple - clean forms with a modern polish
  4. Flexibility without Compromise - the system works on all devices without sacrificing quality
  5. Evolution, not Revolution - the system is adapted to grow along with the company's needs

Spacing

Fixfactor uses multiples of 0.5rem (8px) as the base unit of spacing, ensuring consistency throughout the system.

Spacing Units

XS (8px)
S (16px)
M (24px)
L (32px)
XL (48px)
XXL (80px)
Name Value Application
XS 0.5rem (8px) Smallest spacings, padding for small elements
S 1rem (16px) Standard internal padding, small margins
M 1.5rem (24px) Spacing between related elements
L 2rem (32px) Section spacings, larger margins
XL 3rem (48px) Large spacings between sections
XXL 5rem (80px) Very large spacings (e.g. section padding)

Implementation Guidelines

The following guidelines help maintain consistency and quality when implementing the Design System in Fixfactor projects.

1. Visual Consistency

  • Use only defined colors, spacing, and typography
  • Maintain appropriate element proportions
  • Follow interface rules

2. Accessibility

  • Maintain appropriate contrast (WCAG AA)
  • Provide alternative descriptions for visual elements
  • Ensure proper heading hierarchy

3. Performance

  • Optimize images
  • Minimize the number of animations on mobile pages
  • Use lazy loading for off-screen elements

4. WordPress Implementation

  • Components should be copied to an HTML block in Gutenberg
  • JavaScript should be placed at the end of the HTML block
  • wp_enqueue_scripts can be used for loading code for better performance
